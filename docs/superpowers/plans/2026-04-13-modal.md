## Background: How Radix Dialog works

Radix Dialog renders into a Portal (appended to `document.body`), so it's immune to ancestor CSS stacking contexts. The key primitives:

- `Dialog.Root` — controlled via `open` + `onOpenChange` props
- `Dialog.Portal` — renders children into `document.body`
- `Dialog.Overlay` — the backdrop element (sibling of Content in DOM)
- `Dialog.Content` — the dialog panel; has `role="dialog"` and `aria-modal="true"`; auto-wires `aria-labelledby` to `Dialog.Title`
- `Dialog.Title` — renders as `<h2>`; provides the accessible name
- `Dialog.Close` — renders as `<button>`; closes the dialog on click

Radix closes the dialog automatically when Escape is pressed or a pointer event lands outside `Dialog.Content` (`onPointerDownOutside`). Our `onClose` callback fires via `onOpenChange`.

## Background: Sizing & scroll model

The panel is `Dialog.Content` itself (not a wrapper div inside it):

```
Dialog.Content — fixed top-1/2 left-1/2 -translate-x-1/2 -translate-y-1/2 z-modal
  width:  w-[480px] max-w-[90vw]  (md)  OR  w-[720px] max-w-[90vw]  (lg)
  height: flex flex-col min-h-[240px] max-h-[85vh] overflow-hidden
  
  ├── header (flex-shrink-0, h-14)   — never compresses; title truncates with ellipsis
  └── body (flex-1, overflow-y-auto) — NO padding; scrolls when content overflows
```

**Why no padding on the body:** `ArticleEditor` already has `px-6 py-6` on its form. The modal body is a neutral scroll container — consumers own their padding. Future modal consumers that need padding add it themselves.

**Scroll behaviour with ArticleEditor:** `ArticleEditor`'s form has `h-full overflow-y-auto`. Inside the modal body (`flex-1 overflow-y-auto`), `h-full` equals the modal body's computed height, so the form fills the body exactly. The form scrolls when content overflows — the modal body itself does not. Only one scrollbar is visible at a time. This is correct.

**Title truncation:** The header is a fixed flex row. `Dialog.Title` has `flex-1 truncate` — it takes all available space and clips with `text-overflow: ellipsis`. `Dialog.Close` has `flex-shrink-0 ml-4` — it is never squeezed by a long title.

```tsx
import * as Dialog from '@radix-ui/react-dialog'
import clsx from 'clsx'

export interface ModalProps {
  isOpen: boolean
  onClose: () => void
  title: string
  size?: 'md' | 'lg'
  children: React.ReactNode
}

const sizeClass = {
  md: 'w-[480px]',
  lg: 'w-[720px]',
} as const

export function Modal({ isOpen, onClose, title, size = 'md', children }: ModalProps) {
  return (
    <Dialog.Root open={isOpen} onOpenChange={open => { if (!open) onClose() }}>
      <Dialog.Portal>
        {/* Backdrop */}
        <Dialog.Overlay
          data-testid="modal-overlay"
          className="fixed inset-0 z-modal bg-black/50"
        />

        {/* Panel — Dialog.Content IS the dialog element (role="dialog", aria-modal) */}
        <Dialog.Content
          className={clsx(
            'fixed top-1/2 left-1/2 -translate-x-1/2 -translate-y-1/2 z-modal',
            'flex flex-col min-h-[240px] max-h-[85vh] max-w-[90vw]',
            'bg-surface border border-border rounded-lg shadow-lg overflow-hidden',
            sizeClass[size],
          )}
        >
          {/* Header */}
          <div className="flex items-center h-14 px-6 border-b border-border flex-shrink-0">
            <Dialog.Title className="flex-1 truncate text-base font-semibold text-text-primary">
              {title}
            </Dialog.Title>
            <Dialog.Close
              aria-label="Close"
              className="flex-shrink-0 ml-4 text-text-muted hover:text-text-primary transition-colors duration-fast"
            >
              ✕
            </Dialog.Close>
          </div>

          {/* Body — no padding; consumers own their own padding */}
          <div className="flex-1 overflow-y-auto">
            {children}
          </div>
        </Dialog.Content>
      </Dialog.Portal>
    </Dialog.Root>
  )
}
```

```tsx
function ArticlesPanel() {
  const articles = useArticleStore(state => state.articles)
  const [selectedId, setSelectedId] = useState<string | null>(null)
  const [isEditorOpen, setIsEditorOpen] = useState(false)
  const [editingArticle, setEditingArticle] = useState<Article | undefined>(undefined)

  const selectedArticle = articles.find(a => a.id === selectedId) ?? null

  function handleSelect(article: Article) {
    setSelectedId(article.id)
    setEditingArticle(undefined)
  }

  function handleNew() {
    setEditingArticle(undefined)
    setIsEditorOpen(true)
  }

  function handleEdit(article: Article) {
    setEditingArticle(article)
    setIsEditorOpen(true)
  }

  function handleSaved(article: Article) {
    setSelectedId(article.id)
    setIsEditorOpen(false)
    setEditingArticle(undefined)
  }

  function handleCancel() {
    setIsEditorOpen(false)
    setEditingArticle(undefined)
  }

  function handleNavigateTo(article: Article) {
    setSelectedId(article.id)
    setEditingArticle(undefined)
  }

  return (
    <div className="flex h-full min-h-0">
      {/* Article list — fixed width left column */}
      <div className="w-72 shrink-0 border-r border-border overflow-hidden flex flex-col">
        <ArticleList
          selectedId={selectedId}
          onSelect={handleSelect}
          onNew={handleNew}
        />
      </div>

      {/* Right panel — detail or empty state; editor is now in a modal */}
      <div className="flex-1 min-w-0 overflow-hidden">
        {selectedArticle ? (
          <ArticleDetail
            article={selectedArticle}
            allArticles={articles}
            onBack={() => setSelectedId(null)}
            onEdit={handleEdit}
            onNavigateTo={handleNavigateTo}
          />
        ) : (
          <div className="flex h-full items-center justify-center">
            <p className="text-sm text-text-muted">Select an article to view it.</p>
          </div>
        )}
      </div>

      {/* Editor modal — renders into document.body via Radix Portal */}
      <Modal
        isOpen={isEditorOpen}
        onClose={handleCancel}
        title={editingArticle ? `Edit ${editingArticle.title}` : 'New Article'}
        size="lg"
      >
        <ArticleEditor
          article={editingArticle}
          allArticles={articles}
          onSaved={handleSaved}
          onCancel={handleCancel}
        />
      </Modal>
    </div>
  )
}
```

## Notes for the implementer

**ArticleEditor renders its own title and cancel button.** When displayed inside the Modal, `ArticleEditor`'s internal `<h2>` ("New Article" / "Edit Article") and its top Cancel link appear inside the modal body — below the Modal's own title header. This duplication is intentional for Phase 1 — do not modify `ArticleEditor` internally to remove them.

**Two cancel paths are fine.** The modal ✕ button and ArticleEditor's Cancel button both call `handleCancel`. Both close the modal. This is normal modal UX.

**The `data-testid="modal-overlay"` on `Dialog.Overlay`** is there specifically for the overlay-click test. Do not remove it.
